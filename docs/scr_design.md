# 오늘물줘 MVP 화면 구성 설계안

## **1\. 공통 레이아웃 및 디자인 시스템 (Tailwind CSS)**

* **디자인 원칙**: 모바일 우선(Mobile-First) 구조 (max-w-md mx-auto), Clean & Minimal 식물 케어 테마.  
* **전역 상태/유틸 연동**:  
  * PlantImage: 이미지 로딩 실패 시 /images/plants/placeholder.jpg 자동 폴백 처리.  
  * StatusBadge: overdue(레드: 지연), today(그린: 오늘), upcoming(그레이: 예정) UI 매핑.  
  * GlobalToast: \[개선안 1\] 반영을 위한 전역 스낵바 컴포넌트. "물주기 완료 로그가 추가되었습니다. \[실행 취소\]" 인터랙션 지원.  
  * NavigationGuard: \[개선안 3\] 반영을 위한 커스텀 다이얼로그(얼럿). 폼 작성 중 이탈 시 "작성 중인 내용이 저장되지 않습니다. 나가시겠습니까?" 경고 처리.

## **2\. 홈 화면 (**/**) 설계**

### **2.1 UI/UX 구성 요소**

* **헤더 섹션**: 앱 타이틀 ("오늘물줘") 및 브라우저 로컬 시스템 기준 현재 날짜(YYYY-MM-DD) 출력.  
* **대시보드 요약 (카운터)**: '오늘 물 줄 식물' 및 '물주기 지연' 식물 수를 상단에 배치하여 즉각적인 인지 제공.  
* **메인 콘텐츠 영역 (식물 목록)**:  
  * **식물 데이터가 없는 경우**: EmptyState 컴포넌트 출력 ("등록된 식물이 없습니다. 첫 식물을 등록해보세요\!").  
  * **식물 데이터가 있는 경우**: PlantCard 리스트 배치.  
* **하단 플로팅 액션 버튼 (FAB)**: 식물 등록 화면(/add)으로 이동하는 고정 \[+\] 버튼.

### **2.2 PlantCard 내부 요소 및 인터랙션 (UX 개선)**

* **컴포넌트 구조**: 정적 이미지(PlantImage), 사용자 지정 별명, 식물종 프리셋명, 다음 물주기 날짜(YYYY-MM-DD), 실시간 StatusBadge.  
* **핵심 인터랙션 & 개선 로직**:  
  * **카드 바디 클릭**: 해당 식물의 상세 화면(/plants/:plantId)으로 라우팅.  
  * **물주기 완료 버튼**: 클릭 시 즉시 completeWatering 로직 실행 (마지막 날짜를 오늘로 변경, 차기 날짜 계산, LocalStorage 저장 후 UI 리렌더링).  
  * **\[개선안 1 적용\]**: 물주기 완료 직후 화면 하단에 GlobalToast ("물주기 완료 기록이 추가되었습니다. \[실행 취소\]") 노출. 사용자가 '\[실행 취소\]' 클릭 시 LocalStorage 최신 로그를 롤백하고 이전 날짜 상태로 즉시 복구.

## **3\. 식물 등록 화면 (**/add**) 설계**

### **3.1 UI/UX 구성 요소**

* **상단 내비게이션 (UX 개선)**:  
  1. \[\< 뒤로가기\] 버튼 및 화면 타이틀 ("식물 등록").  
  2. **\[개선안 3 적용\]**: 사용자가 '식물 별명'을 1자 이상 입력했거나 '마지막 물 준 날짜'를 변경한 상태에서 \[\< 뒤로가기\] 터치 시, 즉시 라우팅을 차단하고 NavigationGuard 커스텀 팝업("작성 중인 내용이 저장되지 않습니다. 나가시겠습니까? \[취소\] \[확인\]")을 띄움.  
* **단계별 폼 구성**:  
  1. **인기 식물 Top5 리스트 (**PlantPresetCard**)**: 5종 식물(몬스테라, 스투키, 스킨답서스, 산세베리아, 행운목) 캐러셀 뷰. 선택 시 그린 테두리 하이라이트 및 해당 프리셋의 기본 정보(주기, 양, 방식) 하단에 동적 노출.  
  2. **식물 별명 입력란**: input\[type="text"\] (Placeholder: "예: 거실 몬스테라").  
  3. **마지막 물 준 날짜 선택란**: input\[type="date"\] (기본값: 오늘 날짜).  
* **폼 제출**: "등록하기" 버튼 (검증 통과 시 활성화, 저장 후 / 로 라우팅).

### **3.2 폼 검증(Validation) 규칙**

* **식물 미선택**: 프리셋 ID 부재 시 ➡️ "식물을 선택해주세요." 문구 노출.  
* **별명 미입력**: 공백 제거 후 빈 값일 시 ➡️ "식물 별명을 입력해주세요." 문구 노출.  
* **미래 날짜 입력**: 선택 날짜가 오늘보다 미래일 시 ➡️ "마지막 물 준 날짜는 오늘 이후일 수 없어요." 경고 노출 및 제출 차단.

## **4\. 식물 상세 화면 (**/plants/:plantId**) 설계**

### **4.1 UI/UX 구성 요소**

* **상단 내비게이션 (UX 개선)**:  
  * \[\< 뒤로가기\] 버튼 배치.  
  * **\[개선안 2 적용\]**: 기존 정적 타이틀 "식물 상세"를 제거하고, 데이터 원본에서 가져온 **현재 식물의 별명(예: '거실 몬스테라')을 헤더 타이틀로 동적 바인딩**하여 사용자에게 명확한 콘텍스트 제공.  
* **식물 프로필 섹션**: 대형 크기의 PlantImage 배너, 식물 유형 이름, 현재 물주기 StatusBadge 및 정확한 다음 물주기 예정일 부각.  
* **식물 정보 가이드 (메타 정보 Grid)**: 2열 그리드 레이아웃. 관리 난이도, 물주기 주기, 물주는 양, 물주는 방식, 빛 조건 표시.  
* **텍스트 블록 섹션**: 프리셋 기반 관리 팁(careTip) 및 적색 경고 아이콘 스타일이 가미된 주의사항(caution).  
* **하단 액션 바**:  
  * **물주기 완료 버튼**: 홈 화면과 동일한 완료 로그 생성 및 날짜 갱신 매핑 (클릭 시 하단 타임라인 리스트 즉시 동기화).  
  * **식물 삭제 버튼**: 적색 버튼. 클릭 시 내장 window.confirm 또는 모달을 통해 "정말 삭제하시겠습니까?" 컨펌 후 LocalStorage에서 영구 제거 및 홈(/)으로 강제 라우팅.

### **4.2 물주기 기록 타임라인 (**WateringLog**)**

* **역순 정렬 리스트**: 최신 기록(wateringLogs 배열의 0번째 인덱스)이 최상단에 오도록 타임라인 도트 기호와 함께 세로 정렬 출력.  
* **표시 데이터**: 실제 물 준 날짜 및 기록이 생성된 시간 정보(YYYY-MM-DD HH:mm).  
* **데이터 부재 시**: "아직 물주기 기록이 없습니다." 공백 화면(Empty State) 대체 처리.
