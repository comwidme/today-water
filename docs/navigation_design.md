# 오늘물줘 MVP 네비게이션 설계안

## **1\. 라우트 정의 및 정보 구조 (Information Architecture)**

모바일 브라우저 최적화(max-w-md)를 위해 뎁스(Depth)를 최대 2단계로 제한하여 직관성을 확보합니다.

* / **(홈 화면)** \[Depth 1\]  
  * 앱 진입 시 첫 화면. 대시보드 요약 정보 및 나의 식물 목록 제공.  
* /add **(식물 등록 화면)** \[Depth 2\]  
  * 인기 식물 프리셋 선택 및 신규 식물 추가 폼 제공.  
* /plants/:plantId **(식물 상세 화면)** \[Depth 2\]  
  * 동적 파라미터(:plantId) 기반 개별 식물 메타 정보 및 물주기 타임라인 노출.

## **2\. 사용자 흐름(User Flow) 및 네비게이션 인터랙션**

### **2.1 홈 화면 (**/**)에서의 흐름**

* **식물 카드 바디 영역 클릭**: 해당 식물의 고유 ID를 파라미터로 조합하여 식물 상세 화면(/plants/:plantId)으로 라우팅 이동.  
* **플로팅 액션 버튼** \[+\] **클릭**: 식물 등록 화면(/add)으로 라우팅 이동.  
* **식물 카드 내** \[물주기 완료\] **클릭**:  
  * 페이지 이동 없음 (In-place 업데이트).  
  * 화면 하단에 GlobalToast("물주기 완료 기록이 추가되었습니다. \[실행 취소\]") 생성.  
  * 사용자가 '\[실행 취소\]'를 누르면 데이터 상태를 이전으로 롤백(In-place).

### **2.2 식물 등록 화면 (**/add**)에서의 흐름**

* \[\< 뒤로가기\] **버튼 클릭 (내비게이션 가드 인터셉션)**:  
  * *조건 A (폼 변경 없음)*: 즉시 홈 화면(/)으로 라우팅 복귀.  
  * *조건 B (별명 입력 또는 날짜 변경 있음)*: 라우팅을 일시 차단하고 NavigationGuard 모달("작성 중인 내용이 저장되지 않습니다. 나가시겠습니까? \[취소\] \[확인\]")을 띄움. '\[확인\]' 클릭 시 홈으로 이동, '\[취소\]' 클릭 시 작성 상태 유지.  
* \[등록하기\] **버튼 클릭 (유효성 검증 완료 시 활성화)**:  
  * LocalStorage에 신규 식물 객체 데이터 적재.  
  * 성공 시 홈 화면(/)으로 라우팅 이동 (새 목록 리렌더링).

### **2.3 식물 상세 화면 (**/plants/:plantId**)에서의 흐름**

* **헤더 동적 타이틀 매핑**: 진입 시 헤더 영역에 고정 문구 대신 현재 조회 중인 식물의 별명(예: "거실 몬스테라")을 동적 바인딩하여 콘텍스트 가시성 강화.  
* \[\< 뒤로가기\] **버튼 클릭**: 홈 화면(/)으로 라우팅 복귀.  
* \[물주기 완료\] **버튼 클릭**: 페이지 이동 없이 타임라인 리스트 데이터 및 뱃지 상태 실시간 동기화 (In-place).  
* \[식물 삭제\] **버튼 클릭**:  
  * window.confirm("정말 삭제하시겠습니까?") 트리거.  
  * 사용자 승인 시 LocalStorage에서 해당 ID 데이터 영구 삭제 후 홈 화면(/)으로 강제 라우팅 복귀.

## **3\. 예외 처리 및 브라우저 내비게이션 대응 (GitHub Pages)**

* **존재하지 않는 라우트 (404 예외 처리)**:  
  * 사용자가 잘못된 URL(예: /invalid-path) 또는 삭제된 식물 ID(/plants/deleted-id)로 직접 진입 시, 홈 화면(/)으로 리다이렉트(Redirect) 처리 및 토스트 알림 안내.  
* **브라우저 뒤로가기(PopState) 대응**:  
  * 등록 화면(/add)에서의 뒤로가기 동작 시에도 NavigationGuard 상태 로직이 정상 작동하도록 useEffect 기반의 브라우저 이벤트 리스너 또는 라우터 내장 가드 기능을 매핑함.
